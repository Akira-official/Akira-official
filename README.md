## 🧪 Problem Solving Approach

1. **Understand the actual problem** — not the surface request, the root cause
2. **Map the system** — identify inputs, outputs, failure points
3. **Build the smallest working version** — prove it works before scaling
4. **Iterate with intent** — improve what matters, cut what doesn't
5. **Ship** — done is better than perfect, but never ship broken

---

## 🏗️ Workflow / How I Build
```
Idea → Research → Architecture → Build MVP → Test → Refine → Ship → Improve
```

- I plan before I code. Structure first, syntax second.
- I keep components modular and logic clean from day one
- I document as I build — not as an afterthought
- Every project gets a proper README before it gets pushed

---

## ✍️ Code Style

- **Readable over clever** — code is written once, read many times
- **Minimal dependencies** — only pull in what's actually needed
- **Consistent naming** — clear variable and function names, always
- **Comments for intent** — explain *why*, not *what*
- **Small commits, clear messages** — git history as a dev log
